How It Works

Key Features

Everything you need for comprehensive accessibility auditing

Smart Page Discovery

Finds pages via sitemap.xml, auto-crawls links when no sitemap exists, respects robots.txt, and supports concurrent multi-page scanning.

Dual Viewport + SPA Support

Tests at desktop and mobile viewports. Detects React, Vue, Next.js, Angular and waits for DOM stability. Bypasses Cloudflare challenges automatically.

Confidence Scoring

Every violation tagged as confirmed, likely, or needs review. Low-confidence false positives penalize the score 85% less than confirmed issues.

Keyboard Trap Detection

Simulates Tab cycling to detect focus traps where keyboard users get stuck. Verified with Shift+Tab to eliminate false positives. WCAG 2.1.1 compliant.

Interactive State Scanning

Expands accordions, tabs, and hidden content to find violations inside collapsed elements. Scans popups and overlays before and after dismissal.

Remediation & Reports

WCAG criteria mapping, fix guidance, before/after code, annotated screenshots, and 4 export formats (HTML, PDF, JSON, CSV) with client branding.

Severity Levels

Violations are categorized by impact to help prioritize fixes

Critical

Completely prevents access for some users. Must be fixed immediately to ensure basic usability.

Serious

Creates significant barriers that make content very difficult to access for users with disabilities.

Moderate

Causes some difficulty for users with disabilities but does not completely block access.

Minor

Slightly degrades the user experience but has minimal impact on accessibility.

Quick Start

# Basic scan with HTML report
npm start -- https://example.com

# Full audit with all formats and branding
npm start -- https://example.com -f all --client-name "Acme Corp" --client-color "#ff6600"

# Thorough scan with crawling and interactive states
npm start -- https://example.com --crawl --interactive --concurrency 3 --max-pages 20

Available Options

-f, --formatReport format: json, csv, html, pdf, or all -o, --outputOutput directory (default: ./reports) --client-nameClient name for report branding --client-logoPath to client logo image --client-colorAccent color (hex) -t, --tagsWCAG tags to test (comma-separated) --timeoutPage load timeout in ms (default: 30000) --max-pagesMaximum pages to scan (default: 100) --crawlEnable link-based page discovery (BFS crawl) --interactiveScan interactive states (accordions, tabs, details) --concurrencyPages to scan in parallel (default: 1) --extra-urlsText file with additional URLs (one per line) --axe-configJSON config to disable rules or include/exclude selectors
